# BTF (BPF Type Format)

## Summary
BTF is a compact, efficient metadata format that encodes C-type information (structures, unions, enums, function signatures) used by eBPF CO-RE to enable portable kernel programs. It serves as the bridge between compile-time assumptions and runtime kernel realities.

## Key Points

- **Compact Encoding**: ~10x smaller than DWARF debug information
- **Kernel-Embedded**: Ships with modern kernels at `/sys/kernel/btf/vmlinux`
- **Type Description**: Captures struct layouts, field names, offsets, and types
- **CO-RE Foundation**: Enables runtime field offset calculation and verification
- **Compiler Support**: Generated by Clang/LLVM when compiling eBPF programs

## Details

### What BTF Contains

> [!info] BTF Data Model
> BTF describes the type graph of C programs: structs contain fields with types, which may be other structs, unions, pointers, or primitives. This recursive description allows complete type reconstruction.

BTF captures:

- **Base types**: `int`, `char`, `long`, etc.
- **Derived types**: pointers, arrays, function pointers
- **Struct/Union definitions**: Field names, offsets, sizes
- **Enum definitions**: Named constants and values
- **Function signatures**: Return types and parameter lists

### BTF in the Kernel

Modern kernels (5.2+) embed BTF metadata:

```bash
# Check if BTF is available
$ ls /sys/kernel/btf/vmlinux
/sys/kernel/btf/vmlinux

# Inspect BTF contents
$ bpftool btf dump file /sys/kernel/btf/vmlinux | head -20
```

The kernel exposes its own type information through this interface, allowing loaders to discover the actual memory layout at runtime.

### BTF in Compiled eBPF Objects

When compiling eBPF code:

```c
// Include kernel type definitions
#include "vmlinux.h"

struct task_struct *task = ...;
int pid = task->pid;  // Compiler emits BTF relocation
```

The resulting `.o` file contains:
- eBPF bytecode
- **BTF section**: Type information for all referenced kernel structs
- **Relocation section**: Records of which fields need offset resolution

### BTF vs DWARF Comparison

| Aspect | BTF | DWARF |
|--------|-----|-------|
| **Size** | Kilobytes | Megabytes |
| **Parsing Speed** | Fast (linear) | Slow (complex tree) |
| **Kernel Use** | Yes (embedded) | No (debug only) |
| **eBPF Loader** | Native support | Not used |
| **Information** | Type layout only | Full debug info (line numbers, variables) |

### Generating BTF

To enable BTF in kernel builds:

```bash
# Kernel config
CONFIG_DEBUG_INFO_BTF=y

# Requires pahole tool
# BTF is generated during kernel build from DWARF
```

For eBPF programs, BTF is generated automatically by LLVM when using `-g` (debug info) flags.
