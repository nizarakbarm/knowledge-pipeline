### **Background**
I want to talk about the type of HTTP Flood attack experienced at this page as background and why is not using Cloudflare? There is four reason which are repetitive attacker IP addresses to multiple domain in long time period, target is layer 7 attack especially HTTP to specific domain, burden to local WAF or Flood Protection which leads to high resource usage.

Firstly, we observed repetitive attacker IP addresses targeting multiple domains over a prolonged period, which is a clear indication of a sustained attack strategy. Secondly, the attack was a layer 7 attack, specifically targeting HTTP requests to a specific domain, which adds complexity to the defense mechanisms. Additionally, relying on local Web Application Firewalls (WAF) or Flood Protection systems placed a heavy burden on our resources, leading to high resource usage. The solution is using non local or CDN WAF like cloudflare. But some users consent to changing their nameservers, which can limit flexibility. And Cloudflare's limitations, such as allowing only 5 Web Application Firewall (WAF) rules and a rate-limiting period of just 10 seconds, can be restrictive.
